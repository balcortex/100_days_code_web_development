Day 141 - HTML: Web Forms - Pt. 2

We already saw that the form element is responsible for sending information to the server, but how exactly is the data sent?

The name attribute

Just like we use labels and IDs to refer to an input, the data we send to the backend must also be identified.

We do this by adding a name attribute to our inputs:

<label for="first_name">First Name:</label>
<input type="text" id="first_name" name="first_name">

You can think of it as a variable name for the input. Form input should always have a name attribute; otherwise, it will be ignored when the form is submitted .

To better understand what this looks like, we can submit a form to httpbin. This service will send a response to let us view what data was submitted.

<form action="https://httpbin.org/post" method="post">
 ...
 <input type="text" name="first_name" id="first_name">
 ...
 <input type="text" name="last_name" id="last_name">
 ...
 <input type="number" name="user_age" id="age">
 ...
 <button type="submit">Submit</button>
 ...
</form>

Which is rendered something like this:

User Information

Once we press the submit button, the information sent (and received by the server), correspondent only to the form, is:

"form": {
 "first_name": "John",
 "last_name": "Doe",
 "user_age": "31"
},

As we see, the name attribute is paired with the correspondent information filled by the user. The ID and the name could be different, but the best practice is to use the same value.
