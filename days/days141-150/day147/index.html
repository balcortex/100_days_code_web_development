Styling validations

Until now, the user must click the submit button to know whether the entered data is valid. We can improve this situation by using the valid and invalid pseudo-classes, which will let the user know if the input is valid or not even before he tries to submit the form.

For instance, we can give the valid form controls a green border and the invalid ones a red border in the CSS document:

input:invalid {
 border-color: red;
}
input:valid {
 border-color: green;
}

Now, the user can receive direct feedback by just looking at the borders. Note that on the left, the name and email fields are invalid because they are required, while the age field is green because it is not mandatory to fill it in. On the right, the email is still red as the email is not valid:
