Flexbox Basis

The property flex-basis is used along with flex-grow and flex-shrink to define the width of the flex items. This property is the hardest (and a little tricky) of the three.

Flex-basis is defined as the default size of an element before the extra space is distributed. It behaves similarly to the width property when the items are not allowed to grow or shrink. The default value for flex-basis is auto, which inherits whatever the width value is. However, one difference exists when both properties have the value of 0, as we explore in the following examples.

In the first example, the items cannot grow or shrink. With flex-basis: 0px, the width of Item 1 is reduced to the maximum, allowing the content to display; while setting the width: 0px on Item 2, the box's width is 0px, which makes the content disappear. There's no difference with values greater than the content width, as Item 3 and Item 4 behave the same.

Finally, items are allowed to grow and shrink in the last example. Item 1 has a value of 0px for flex-basis, while items 2 and 3 have values of 180px and auto, respectively. In this configuration, Item 2 will grow until its width is 180px, then Item 3 will start to grow until its width matches the content, and finally, the extra space will be divided equally between the three items.
