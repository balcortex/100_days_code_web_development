Day 54 - JavaScript: Debugging

There are (many) times when the code does not work as we expect, so we must fix the bug (i.e., error). The debugger is a useful tool included in the web browser which allow us to pause the code at any time by the use of so called breakpoints. While the code is paused, we can inspect the variables, and even use the console to input JavaScript statements.

There are a few ways to set a breakpoint, the easiest one is to use the debugger keyword before the line of code we would like to inspect:

result = num1 + num2;
debugger; // The program pauses here
output.textContent = `The sum is: ${result}`;

In the debugger, we can see that the variables num1 and num2 are strings (they are enclosed in quotes) instead of numbers, which cause the malfunction in the program. We can solve this by casting them to numbers, using the plus operator.
