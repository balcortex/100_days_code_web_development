Day 151 - CSS: Grid - Pt. 4

Implicit grid - rows

We already saw that rows are automatically created if more items fit than those declared in the grid-template properties. For instance, suppose that we have a 4-item container with the following properties (note the grid-auto-rows property that we will cover later):

.container {
 display: grid;
 grid-template-columns: 50px 80px;
 grid-template-rows: 50px 100px;
 grid-auto-rows: 20px;
}

As the grid-template properties declare, the layout is set as a 2x2 grid, perfect for the 4-item container. But what happens if we add another item without changing the grid-template properties:

<div class="container">
 <div>Item 1</div>
 ...
 <div>Item 5</div>
 </div>

The new item (Item 5) is slotted into a third row we did not define. This is because of the implicit grid concept, and it's how CSS Grid can automatically place grid items when we haven't explicitly defined the layout for them.

The size values established from the grid-template-rows property are not carried over into these implicit grid tracks. But we can define values for the implicit grid tracks using the grid-auto-rows property (later, we will see the equivalent for columns). In this way, we can ensure that any new tracks the implicit grid makes for extra content are set at values we defined.
