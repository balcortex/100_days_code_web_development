Day 99 - JavaScript Objects - Pt. 4

Test for existence

Unlike other programming languages, JavaScript allows access to non-existing object properties without throwing any error. If the property doesn't exist, the return will be `undefined`:

let user = {
 forename: 'John',
};

console.log(user.surname); // undefined

Then, to check for existence, you can compare the property's value with `undefined`:

console.log(user.forename === undefined); // false (it does exist)
console.log(user.address === undefined); // true (it doesn't exist)

Also, you can use the `in` operator to check for existence:

let key = 'address';

console.log('forename' in user) // true (it does exist)
console.log(key in user) // false (it doesn't exist)

There is a particular case when checking for equality with `undefined` will return true despite the existence of the property. You can avoid this issue by using the `in` operator instead:

user.address = undefined;

console.log(user.address === undefined) // true (but the key does exist)
console.log('address' in user) // true (it does exist, but it's undefined)

Note: you should never assign `undefined` explicitly to avoid this type of edge case.

Loop over the keys

To loop over all the keys in the object, use the `for ... in` loop:

for (key in object) {
 // executes the body for each key among object properties
}

Following the previous example:

for (let key in user) {
 console.log(key); // forename, address
 console.log(user[key]); // John, undefined
}
