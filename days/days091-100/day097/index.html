Day 97 - JavaScript Objects - Pt. 2

Square bracket notation

You can also define multi-word keys by using quotes at creation time, but rather than use the dot notation to access the value, the square bracket notation has to be used instead:

let user = {
 name: "John", // key `name` stores value "John"
 age: 30, // key `age` stores value 30
 "likes dogs": true, // key `likes dogs` stores true
};
console.log(user["likes dogs"]); // true

The square bracket notation is similar to the dot notation but is more flexible. With the former, you can define a variable that contains the key as its value and use the variable itself to provide the key to the object:

let key = 'name';
console.log(user[key]); // John

But this doesn't work with the dot notation:

console.log(user.key); // undefined

The square brackets can also be used in the object's keys at creation time, which is known as `computed properties`. For instance, you want to use the value of a variable as the key of the object:

let fruit = 'apple';
let shoppingList = {
 fruit: 5,
};
console.log(shoppingList); // Object { fruit: 5 }

That didn't work. We wanted to use `apple` as the key, but instead, we got `fruit` as the key. Let's fix that using the square brackets in the key:

let newShoppingList = {
 [fruit]: 5,
};
console.log(newShoppingList); // Object { apple: 5 }

Square brackets are much more powerful than dot notation. They allow any property names and variables. But they are also more cumbersome to write.

So most of the time, when property names are known and simple, the dot is used. And if we need something more complex, we switch to square brackets.
