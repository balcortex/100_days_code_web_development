Day 98 - JavaScript Objects - Pt. 3

Property value shorthand

Often, we want to use variable names as the property names (i.e., keys), for instance:

function makeUser(name, age) {
 return {
  name: name,
  age: age,
 };
}

let john = makeUser('John', 30);
console.log(john); // Object { name: "John", age: 30 }

In the example above, properties have the same name as variables. The same can also be written as:

function makeUserShorthand(name, age) {
 return {
  name, // same as name: name
  age, // same as age, age
 };
}

let alex = makeUserShorthand('Alex', 25);
console.log(alex); // Object { name: "Alex", age: 25 }

Note: a mixture of standard and shorthand properties can be used.

Property names limitations

Apart from valid variable names (i.e., valid combinations of text, numbers and symbols), property names can also use language-reserved words like `for`, `let`, `return`, etc.

// these properties are all right
let obj = {
 for: 1,
 let: 2,
 return: 3,
 _$3: 4,
};

console.log(obj.for + obj.let + obj.return + obj._$3); // 10

This functionality is allowed because all the keys are converted to strings:

let obj2 = {
 0: 'Zero', // same as '0': 'Zero'
};

console.log(obj2[0]); // Zero
console.log(obj2['0']); // Zero

Note: you cannot access properties that start with digits through dot notation.
