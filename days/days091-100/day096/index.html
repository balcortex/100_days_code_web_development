Day 96 - JavaScript Objects - Pt. 1

Objects are the eighth data types, and the more complex one. The first seventh data types (i.e., primitives) can only store a single thing in their values, whatever it is a number or a string. In contrast, objects can store keyed collections of data.

Objects are widely used in JavaScript, so let's explore the basics first.

An (empty) object can be created with curly brackets (i.e., the object literal syntax) or through the object constructor:

let user = {}; // "object literal" syntax, most common
let user = new Object(); // "object constructor" syntax

console.log(user); // Object { }

The object properties can also be included at creation time:

let user = {
 name: "John", // key `name` stores value "John"
 age: 30, // key `age` stores value 30
};

console.log(user); // Object { name: "John", age: 30 }

The properties are accessible using the dot notation:

console.log(user.name) // John
console.log(user.age) // 30

After creation, new properties can be added using the same dot notation:

user.isAdmin = true;

console.log(user); // Object { name: "John", age: 30, isAdmin: true }

To delete properties, use the `delete` operator:

delete user.age;

console.log(user); // Object { name: "John", isAdmin: true }
