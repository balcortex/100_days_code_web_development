Declaring methods in the prototype

We can declare new methods in the prototype, and all objects inherited from it will have access to them. For instance, look at the following code:

function User(name, isAdmin) {
 this.name = name;
 this.isAdmin = isAdmin;
 this.revokeAdmin = function () {
  this.isAdmin = false;
 };
}

User.prototype.greet = function () {
 console.log(`Hi! I'm ${this.name}`);
};

const user1 = new User("John", false);
const user2 = new User("Andy", true);

user1.greet(); // Hi! I'm John
user2.greet(); // Hi! I'm Andy

Both users can access the method greet() defined in the prototype. But what's the difference between declaring the method in the prototype and the method in the constructor? Let's use the console to examine one of the users:

console.log(user1); // User {name: 'John', isAdmin: false, revokeAdmin: Æ’}

If you look closely, the method greet() isn't declared in `user1`, whereas the method revokeAdmin() is. That means constructor methods are copied in each instance. In contrast, prototype methods are only defined once, which can help us save memory space.
