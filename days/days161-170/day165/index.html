Day 165 - JavaScript: The Prototype

The prototype

All objects in JavaScript have a prototype. The prototype is another object that the original object inherits from, which means that the original object has access to all of its prototype's methods and properties.

Accessing the object's prototype

You can check the object's prototype by using the Object.getPrototypeOf() function on the object:

function User(name, isAdmin) {
 this.name = name;
 this.isAdmin = isAdmin;
}

const user1 = new User("John", false);
const user2 = new User("Andy", true);

console.log(Object.getPrototypeOf(user1)); // Object {...}

console.log(Object.getPrototypeOf(user1) === Object.getPrototypeOf(user2)); // true

From the previous code, we can see that the prototypes of both users are the same. That is, `user1` and `user2` have access to all the methods and properties defined in the `User` prototype.

There's also another way to access the prototype by using the property .prototype on the constructor:

console.log(Object.getPrototypeOf(user1) === User.prototype); // true
