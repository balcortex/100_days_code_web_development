auto-fit vs. auto-fill

On the last day, we saw that we could use the auto-fit (or auto-fill) value on the repeat function to set a flexible number of columns in the container, depending on the container's width. For this task, we used a static column width.

But, the real magic of auto-fill and auto-fit comes when we incorporate the minmax() with dynamic sizes in the equation, adjusting the number of the columns and resizing the items to fill the entire container without overflowing it. This adaptability ensures our layouts remain consistent and visually appealing, regardless of the container's dimensions.

.container {
 display: grid;
 grid-template-rows: repeat(2, 1fr);
 grid-template-columns: repeat(auto-fit, minmax(50px, 1fr));
}

The items will stretch to the container's width, adding or removing columns as the container changes size.

Now, let's explore the difference between auto-fit and auto-fill. In more cases, these two values will behave similarly. The only difference is when the items don't fill an entire row once. While auto-fit will stretch the items in any condition, auto-fill will set the items' width at their minimum once the space becomes available to add another item:

.container {
 grid-template-columns: repeat(auto-fill, minmax(50px, 1fr));
}
