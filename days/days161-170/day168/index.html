Prototypal inheritance

We've seen that when a method is called, JavaScript first searches for it in the current object. If not found, it searches in the object's prototype. The search ends when the method is encountered or if the prototype of the Object.prototype (which is null) is reached.

Since every prototype inherits from Object.prototype by default, and any object can have one unique prototype, how can we create a deeper prototype chain? JavaScripts provides the Object.setPrototypeOf() method for this purpose.

Let's explain this directly in code:

function Person(name) {
 this.name = name;
}

Person.prototype.greet = function () {
 console.log(`Hi I'm ${this.name}`);
};

function User(name, isAdmin) {
 this.name = name;
 this.isAdmin = isAdmin;
}

Object.setPrototypeOf(User.prototype, Person.prototype);

let user1 = new User("John", false);

user1.greet(); // Hi I'm John

The `user1` object can access the methods in the Person.prototype because we make User inherits from Person. The method resolution order for this code would be:

  1. user1
  2. User.prototype
  3. Person.prototype
  4. Object.prototype
