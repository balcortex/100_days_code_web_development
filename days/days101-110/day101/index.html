Day 101 - JavaScript Objects - Pt. 6

Objects within objects

Object properties can be anything, even objects. To access an object property within an object, you have to chain the dot notation (or square bracket notation). For instance:

const user = {
 name: {
  first: 'John',
  last: 'Smith',
 },
 age: 32,
};

console.log(user.name.first) // John

Object methods

You can also define functions as object properties, which are called object methods:

const user = {
 hello: function () {
   console.log('Hello');
 },
};

user.hello(); // Hello

There's also a simplified notation to define methods, in which the function name is taken as the property name:

const user = {
 goodbye () {
   console.log('Bye');
 },
};

user.goodbye(); // Bye

Using object properties within the methods (this)

To refer to the object itself within the methods, use the keyword `this`. For instance:

const user = {
 name: {
  first: 'John',
  last: 'Smith',
 },
 age: 32,
 info: function () {
  console.log(`${this.name.first} ${this.name.last} is ${this.age} years old`);
 },
 greet() {
  console.log(`Hi! I'm ${this.name.first}`);
 },
};

user.info(); // John Smith is 32 years old
user.greet(); // Hi! I'm John

In this case `this` refers to `user`, so you could use the later instead. But, if for some reason you rename the variable `user`, the code will not work without rename the variables inside too. Furthermore, when using object constructors, the use of `this` is (almost) mandatory.
