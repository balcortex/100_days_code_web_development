Day 102 - JavaScript Objects - Pt. 7

Constructors

Using object literals is fine when you only need to create one object. Still, when you need to create multiple ones of the same type, they are inadequate because you must repeat the same code and remember to update the values.

For instance, imagine that you need to create different users with the same set of methods and properties, differing in the properties values. The first option that would come to your mind is to write a function that instantiates and returns the object:

function createPerson(name) {
 let person = {
  name,
  greet() {
   console.log(`Hi! I'm ${this.name}`);
  },
 };
 return person;
}

Then, you can call the function with different arguments to create different users:

const john = createPerson('John');
john.greet(); // Hi! I'm John

const alex = createPerson('Alex');
alex.greet(); // Hi! I'm Alex

This code works fine, but we can do better by using a constructor. A constructor is a function that:

  1. Create a new object
  2. Bind `this` to the new object
  3. Run the code in the constructor
  4. Return the object

Steps 1, 2, and 4 happen implicitly, so you only need to worry about the object's behavior (i.e., the code in the constructor).

By convention, constructors start with a capital letter and are named for the type of object they create. To call a function as a constructor, use the keyword `new`:

function Person(name) {
 this.name = name;
 this.greet = function () {
  console.log(`Hi! I'm ${this.name}`);
 };
}

const jacob = new Person('Jacob');
jacob.greet(); // Hi! I'm Jacob

Constructors are briefer and more straightforward to understand, so they're preferred over functions.
