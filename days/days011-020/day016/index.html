Standard vs. Alternative Box Model

To add (even) more complexity to the Box Model, there are two types of models (i.e., how the box size is calculated):

  • Standard Box Model: Used by default by all browsers, the width and height of the block only account for the content box. For example, if we define a Standard Box Model of 350px by 150px with a padding of 25px and a border of 5px, the actual size of the box would be 260px wide (200 + 25 + 25 + 5 + 5) and 180px high (120 + 25 + 25 + 5 + 5)
    • Standard Box Model (default) Size 260px by 180px
  • Alternative Box Model: The content, the padding, and the border add up to the total size of the box. Following the previous example, the actual size of the box would be 200px wide (60 padding and border + 120 content width) and 120px high (60 padding and border + 60 content height).
    • Alternate Box Model Size 200px by 120px

Note: the margin doesn't add up space for the box.

The Alternative Box Model is easier to work with, and most developers commonly use it.
