Day 129 - CSS: Advanced Selectors - Pt. 4

Attribute selectors

Recall that an attribute is anything in the opening tag of an HTML element - such as class='my-class' or src='pic.jpg'.

Since these values are very user-defined, we need a more flexible approach to target specific values. First, let's look at the basic usage:

  • [attribute] - This will select anything where the given attribute exists, despite its value.
  • selector[attribute] - Optionally, we can combine the attribute selectors with other types of selectors, like class or element selectors.
  • [attribute="value"] - To get specific, we can use the = to match a particular attribute with a specific value.

Now, let's see some real cases:

[src] {
 /* This will target any element that has a src attribute. */
}

img[src] {
 /* This will only target img elements that have a src attribute. */
}

img[src="puppy.jpg"] {
 /* Elements with a src attribute that is exactly "puppy.jpg" */
}

Sometimes, we must match a part of the attribute's value to target several elements simultaneously. For this purpose, we can use the following syntax:

  • [attribute^="value"] - ^= Will match strings from the start.
  • [attribute$="value"] - $= Will match strings from the end.
  • [attribute*="value"] - *= The wildcard selector will match anywhere inside the string

For instance:

[class^='aus'] {
 /* Classes are attributes too!
 This will target any class that begins with 'aus':
 class='austria'
 class='australia'
 */
}

[src$='.jpg'] {
 /* This will target any src attribute that ends in '.jpg':
 src='puppy.jpg'
 src='kitten.jpg'
 */
}

[for*='ill'] {
 /* This will target any for attribute that has 'ill' anywhere inside it:
 for="bill"
 for="jill"
 for="silly"
 for="ill"
 */
}
