Day 126 - CSS: Advanced Selectors - Pt. 1

Child and sibling combinators

There are several ways to access different elements without using their id or class. Following are three new selectors to do that:

  • > : the child combinator
  • + : the adjacent sibling combinator
  • ~ : the general sibling combinator

To see how they work, we are going to be using the following HTML code:

<main class="parent">
 <div class="child group1">
  <div class="grand-child group1"></div>
 </div>
 <div class="child group2">
  <div class="grand-child group2"></div>
 </div>
 <div class="child group3">
  <div class="grand-child group3"></div>
 </div>
</main>

You should know that to select all the divs inside main (including child and grandchild), you would write:

main div {
 /* CSS code */
}

Child combinator

The child selector will select an element that is one level of indentation down. For instance, to select only the divs with a class of child, you can use:

main > div {
 /* CSS code */
}

To select only the grandchildren (without the children):

main > div > div {
 /* CSS code */
}

Adjacent sibling combinator

We can use the adjacent sibling combinator to select an adjacent element (immediately following) to our target or on the same level of indentation.

For instance, to select the class `child group2` you can use:

.group1 + div {
 /* CSS code */
}

And, to select the `child group3`, add another combinator:

.group1 + div + div {
 /* CSS code */
}

General sibling combinator

If you want to select all the siblings, use the general sibling combinator. For instance, you can select all the elements with the class `child` (including groups 1, 2, and 3) using:

.group1 ~ div {
 /* CSS code */
}
