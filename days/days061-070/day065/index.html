Day 65 - JavaScript: DOM Manipulation, Pt. 1

One of the most valuable abilities of JavaScript is the ability to manipulate the DOM (Document Object Model), which is a tree-like representation of the contents of a webpage. The DOM models the relationship of elements depending on how they're arranged in the HTML document.

Suppose you have the following HTML document:

<div id="container">
 <div class="header"></div>
 <div class="main"></div>
 <div class="footer"></div>
</div>

The `container` element has three children. Thus, `container` is a parent of `header`, main is child of `container`; and `footer`, `main` and `header` are siblings. Graphically, siblings have the same indentation level, while children are separated by one indentation from their parents.

container
 header
 main
 footer

There are several ways to target (i.e., refer) an element on JavaScript. One approach to target an element is to use the document.querySelector() method, which returns a reference (or an array of references) of the match. Once the reference is obtained, you can use several methods to manipulate the element. For example, let's target the `container` and investigate how many children and what his children are:

const container = document.querySelector("#container");
console.log(container.childElementCount); // 3
console.log(container.children);
// HTMLCollection(3) [div.header, div.main, div.footer]

You can select any child using its corresponding index:

console.log(container.children[0]); // div.header
console.log(container.children[1]); // div.main
console.log(container.children[2]); // div.footer

After targeting the element, you can modify its attributes, including its content and style. We will see how in the following days.
