Day 64 - JavaScript: Functions, Pt. 4

There is yet another feature of function expressions: arrow functions. These functions are a straightforward and concise way of creating functions.

First, let's remember how function expressions are declared:

let mySum = function (a, b) {
 return a + b;
}
console.log(mySum(1, 2)); // 3

Using the arrow syntax, the previous function is expressed as:

let mySum2 = (a, b) => {
 let result = a + b;
 return result;
};
console.log(mySum2(1, 2)); // 3

This code could look very similar, but the intermediate variable result was solely used to explain what a multiline arrow function looks like. If the function is concise and can be expressed in one line, the function would look like this:

let mySum3 = (a, b) => a + b;
console.log(mySum3(1, 2)); // 3

Even more, if the function only receives one argument, the parentheses can be omitted:

let mySquare = a => a * a;
console.log(mySquare(3)); // 9

If the function receives no arguments, the parentheses are left blank but must be included:

let greet = () => console.log('Hello');
greet(); // Hello

Arrow functions are clear and concise and often are the best choice if the function is simple. However, if the function is rather complex, use function declaration instead.
