Day 62 - JavaScript: Functions, Pt. 2

There are two approaches to defining a function:

  • Function definition
  • Function expression

Using definition syntax is the most common way to create a function:

function greetWorldDef(){
 console.log('Hello World (definition)');
}
greetWorldDef(); // Hello World (definition)

The other approach is known as a function expression, in which the function's name is declared as a variable using the keyword let. Then, the actual function is assigned next:

let greetWorldExp = function(){
 console.log('Hello World (expression)');
};
greetWorldExp(); // Hello World (expression)

Both functions behave similarly. However, the main difference is that functions created using definition can be used before the actual code when it is defined:

functionBeforeDef(); // This works
function functionBeforeDef(){
 console.log('This works');
}

Trying the same with function expressions does not work:

functionBeforeExp(); // ReferenceError
let functionBeforeExp = function(){
 console.log('This code will not be executed anyway');
};

As a rule of thumb, you should always use function definition to create functions. However, function expressions have one advantage over definitions, and we will discover it tomorrow.
