Day 63 - JavaScript: Functions, Pt. 3

Function definitions should always be used over function expressions, as the code is more explicit, and the function could be used even before the place of definition.

However, function definition must have a name, while function expressions can be unnamed:

let x = function named() {
 console.log('Named function');
};
x(); // Named function

let y = function () {
 console.log('Unnamed function');
};
y(); // Unnamed function

This characteristic is also known as an anonymous function. Anonymous functions are helpful to pass functions as an argument, and the sole use of this function is to be used there and not anywhere else in the code:

function greet(func) {
 func();
}
greet(function () {
 console.log('Hello World!');
}); // 'Hello World!'

Also, another interesting feature of function expressions is that they can be used immediately upon creation, which is known as IIFE ( Immediately Invoked Function Expression):

(function () {
 console.log('IIFE'); // IIFE
})();

Function expression has some interesting characteristics, but when in doubt, use function definition.
