Day 61 - JavaScript: Functions, Pt. 1

In JavaScript, functions are like values but are more like special values. If we define a function and print its value, we get exactly the code used to create the function:

function greetWorld(){
 console.log('Hello World!');
}
console.log(greetWorld);
// function greetWorld(){
//  console.log('Hello World!');
// }

However, the type of a Function is function:

console.log(typeof greetWorld); // function

You can create a copy of the function and store it into another variable:

let greetCopy = greetWorld;
greetCopy(); // Hello World!
console.log(greetWorld === greetCopy); // true

Functions can receive arguments and have a default value if one is not provided:

function greetUser(user = 'Bort'){
 console.log('Hello' + ' ' + user);
}

greetUser() // Hello Bort
greetUser('Homer') // Hello Homer

And, because functions behave like values, you can pass functions as arguments to other functions to be executed within:

function greet(func1, func2, user){
func1(); // Execute here the func1
func2(user); // Execute here the func2
}

greet(greetWorld, greetUser, 'Elon');
// Hello World!
// Hello Elon
