Day 117 - CSS Units Pt. 1

You can use many different units to define sizes in CSS: `px`, `cm`, `em`, `rem`, `%`, etc. These can be divided into two broader classifications:

  • Absolute Units
  • Relative Units

Absolute Units

Absolute units remain the same in any context. For instance, `px` is an absolute unit because the size of a pixel doesn't change relative to anything else on the page. Other absolute units relate to physical units such as in (inch) and cm (centimeter).

You can specify units in several elements; the most common one is font size. Let's see how different values affect the size of the text in the items of a list:

<ul>
 <li style="font-size: 16px;">Font size of 16px (default)</li>
 <li style="font-size: 10px;">Font size of 10px</li>
 <li style="font-size: 24px;">Font size of 24px</li>
</ul>

Which, depending on the browser, is rendered something like:

  • Font size of 16px (default)
  • Font size of 10px
  • Font size of 24px

If you are preparing a document to be printed, you may want to use `cm` or `in` as the units:

<ul>
 <li style="font-size: 16px;">Font size of 16px (default)</li>
 <li style="font-size: 1cm;">Font size of 1cm</li>
 <li style="font-size: 0.3in;">Font size of 0.3in</li>
</ul>

Which, on the screen, would look like this:

  • Font size of 16px (default)
  • Font size of 1cm
  • Font size of 0.3in

Absolute units work great if you want the size to stay the same no matter what. The problem with this approach is that, on bigger screens with high resolution (1920x1080, Full HD), a pixel would look tiny compared to a small screen with a resolution of 100x100. This way, relative units are great for scale content depending on the screen (resolution) size.
