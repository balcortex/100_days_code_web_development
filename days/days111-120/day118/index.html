Day 118 - CSS Units Pt. 2

Relative units

Relative units are units that can change based on their context. As their name suggests, these units are relative to something else, perhaps the size of the parent element's font or the viewport's size.

The benefit of using relative units is that the size of text or other elements scales relative to everything else on the page. That is, the size of an element will adjust whether the user has a big or small screen (it does not depend on the resolution, like the pixel, for example).

There are several relative units, like `em`, `rem`, `vw`, `vh`, etc. Let's start with the `em` unit.

em unit

`em` refers primarily to the font size, though it could be used to define other attributes.

`1em` is the font size of the element's parent in the case of typographical properties like font size, and the font size of the element itself in the case of other properties like width.

For instance, suppose that you have three nested divs: the parent, the child, and the grandchild (child of the child), as the following code shows:

<div>
 Parent
 <div>
  Child
  <div>
   Grandchild
  </div>
 </div>
</div>

If the parent has a fixed size of 16px, and the child has a size of 1.5em, that means that the font size of the child would be 1.5*16 = 24px. Now, the grandchild has a font size of 0.5em, and its parent element is the child, whose font is 24px. That means that the font size of the grandchild would be 12px.

Parent (16px)
Child (1.5em = 1.5*16px = 24px)
Grandchild (0.5em = 0.5*24px = 12px)

Similarly, if both the child and the grandchild have a font size of 2em, that means that the child would be twice as big as the parent, and the grandchild twice as big as the child:

Parent (10px)
Child (2em = 2*10px = 20px)
Grandchild (2em = 2*20px = 40px)
