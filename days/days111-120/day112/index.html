Day 112 - HTML Tables

A table is a structured set of data made up of rows and columns, which is known as tabular data. For instance, a table can show the relationship between a person's forename, surname, and age.

Tables were designed exclusively for use with tabular data, which is the only recommended use. Unfortunately, before the standardization of CSS, many people used HTML tables to lay out web pages. However, this is not recommended nowadays, and you should avoid it.

To declare a table, first use the `table` tag:

<table></table>

The smallest container inside a table is the cell, which is created by the tag `td` (td stands for table data). The following code will create a single cell:

<table>
 <td>This is a single cell</td>
</table>

Which is rendered like this:

This is a single cell

That wasn't the expected result, but tables have no style (no borders, no background, etc.) right out of the box, so you must add the style to make it look like a proper table. Let's add a border to the td selector on the CSS document:

td {
 border: 1px solid black;
}

Now, the cell should look like a proper cell:

This is a single cell

Let's add another cell and see what happens:

<table>
 <td>This is a single cell</td>
 <td>This is another single cell</td>
</table>

Which results in:

This is a single cell This is another cell

Thus, cells are placed right next to each other. To add a cell below, first use the `tr` (tr stands for table row) tag:

<table>
 <tr>
  <td>Cell 1, Row 1</td>
  <td>Cell 2, Row 1</td>
 </tr>
 <tr>
  <td>Cell 1, Row 2</td>
  <td>Cell 2, Row 2</td>
 </tr>
</table>

Which is rendered like this:

Cell 1, Row 1 Cell 2, Row 1
Cell 1, Row 2 Cell 2, Row 2
