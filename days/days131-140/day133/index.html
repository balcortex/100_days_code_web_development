Day 133 - CSS: Variables - Pt. 2

Variable scope

The scope of a custom property includes the selector in which it was defined and any of its descendants. For example, look at the following HTML code:

<div class='container'>
 <p class='par-1'></p>
</div>

<p class='par-2'></p>

And the corresponding CSS code:

.container {
 --main-bg: red;
}

.par-1 {
 background-color: var(--main-bg);
}

.par-2 {
 background-color: var(--main-bg);
}

In this example, only the par-1 class will have its background changed, as the par-2 is not in the scope, so no color will be applied to it (unless a fallback value is used).

Global variables

While you may want to limit the scope of a custom property, the ideal situation is to use them across the entire document. Hence, it is widely used to declare custom properties on the :root selector:

:root {
 --main-bg: red;
}

.par-1 {
 background-color: var(--main-bg);
}

.par-2 {
 background-color: var(--main-bg);
}

By declaring our custom property on the :root selector in the example above, we can access it on any other selector within our CSS file, since any other selector would be considered a descendant of the :root selector.
