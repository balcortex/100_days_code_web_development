Day 132 - CSS: Variables - Pt. 1

Custom properties (variables) allow us to reference a CSS value several times throughout the file. Then, instead of updating every instance, we only have to update the variable itself.

The main advantage of using custom properties is keeping colors consistent throughout a project. Furthermore, we can even redefine the same properties under different contexts, for instance, dark and light themes.

Declaring a custom property

To declare a custom property, use a double hyphen:

.fancy {
 --color-text: black;
}

Using a custom property

To access a custom property, use the var() function:

.fancy {
 ...
 color: var(--color-text);
}

Fallback values

The var() function accepts another optional parameter: a fallback value. If the first value (the custom property) doesn't exist or is invalid, the fallback value is used instead.

.fallback {
 --color-text: black;

 background-color: var(--this-property-does-not-exist, white);
}

In the last example, the background color will be set to white, as the custom property isn't declared.
